Consolidação Actual vs Plan
Q4 2025

Case técnico desenvolvido para o cargo de Analista de Dados na Unilever : pipeline em Python para consolidação financeira de duas subsidiárias, com harmonização de SKU → BFPP, conversão cambial para EUR e análise de gaps executivos.

Período Out – Dez 2025
Output 43.354 linhas
Linguagem Python · Pandas
// 01 · contexto

O desafio

O objetivo foi apresentar ao time de Digital Finance LATAM a consolidação de bases de dados heterogêneas, propiciando uma visibilidade executiva sobre o atingimento de metas e a identificação de desvios financeiros ao longo do Q4 de 2025.

DESAFIO / 01
Harmonização de Granularidade
Actual vem em nível SKU (PKEY individual). Plan está em nível BFPP (agrupamento Brand-Format-Price-Point). Necessário mapear cada SKU ao seu BFPP correspondente via tabela mestre de produto.
DESAFIO / 02
Conversão Cambial
Dois países com moedas locais diferentes. Company identificada pelos 4 primeiros dígitos do COCO. Conversão para EUR usando taxas mensais da tabela currency_df.
DESAFIO / 03
Alinhamento Financeiro
Itens financeiros (FSITMS) precisam ser mapeados ao nível hierárquico L01 para comparação homogênea entre Actual e Plan, com rastreamento de contas não mapeadas.
DESAFIO / 04
Modelagem Analítica
Output estruturado por Company · Mês · BFPP · L01, com outer join para capturar execução sem orçamento e orçamento sem execução, calculando Gap absoluto (EUR) e percentual.

🔗 Estratégia de Join

Outer join entre Actual agregado e Plan agregado garante que nenhum cenário seja perdido: orçamento sem execução (Actual = 0) e execução sem orçamento (Plan = 0) são ambos capturados e rastreáveis.

⚠️ Tratamento de Inconsistências

SKUs sem mapeamento BFPP recebem flag "UNMAPPED_BFPP". Contas sem L01 ficam como "UNMAPPED_L01". Registros sem taxa de câmbio são removidos para evitar distorção financeira.

📐 Granularidade Final

Output estruturado em 4 dimensões: Company · Mês · BFPP · L01. Permite drill-down executivo por país, período, portfólio de produto e linha de conta financeira.

📊 Cálculo de Desvios

Gap_EUR = Actual − Plan. Gap percentual calculado apenas quando Plan ≠ 0 (NaN caso contrário), evitando divisão por zero e métricas enganosas.

// 03 · implementação

Trechos-Chave do Código

filtro_q4_company.py
# Filtro temporal — apenas Q4 2025
meses_q4 = [10, 11, 12]

actual_df = actual_df[
    (actual_df["year"] == 2025) &
    (actual_df["month"].isin(meses_q4))
].copy()

# Extração de Company (4 primeiros dígitos do COCO)
actual_df["Company"] = actual_df["COCO"].astype(str).str[:4]
plan_df["Company"]   = plan_df["COCO"].astype(str).str[:4]

Cópia explícita com .copy() evita SettingWithCopyWarning do Pandas e garante isolamento do dataframe filtrado. Extração de substring por indexing é mais performática que regex para strings curtas e uniformes como COCO.

harmonizacao_bfpp.py
# Actual está no nível SKU → mapear para BFPP
actual_df = actual_df.merge(
    product_df[["PKEY", "BFPP_Code"]],
    on="PKEY",
    how="left"
)

# SKUs sem cadastro recebem flag rastreável
actual_df["BFPP"] = actual_df["BFPP_Code"].fillna("UNMAPPED_BFPP")
unmapped = actual_df["BFPP_Code"].isna().sum()
print(f"SKUs sem mapeamento BFPP: {unmapped}")

# Plan já está em nível BFPP — renomeio apenas para consistência
plan_df["BFPP"] = plan_df["PKEY"]

Left join preserva todos os registros Actual, mesmo sem correspondência no produto mestre. A flag "UNMAPPED_BFPP" em vez de drop silencioso garante auditabilidade dos dados e permite ao time de dados identificar lacunas no cadastro de produtos.

conversao_cambial.py
# Chaves de join adaptativas: considera granularidade de currency_df
fx_keys = ["Company"]

if "year" in currency_df.columns:
    fx_keys.append("year")
if "month" in currency_df.columns:
    fx_keys.append("month")

actual_df = actual_df.merge(
    currency_df[fx_keys + ["Euro_Rate"]],
    on=fx_keys, how="left"
)

# Remove registros sem taxa (evita distorção financeira)
actual_df = actual_df.dropna(subset=["Euro_Rate"])

# Conversão: valor local ÷ taxa EUR
actual_df["Actual_EUR"] = actual_df["TOTAL"] / actual_df["Euro_Rate"]
plan_df["Plan_EUR"]     = plan_df["TOTALEU"]  # Plan já em EUR

As chaves de join FX são dinâmicas: se a tabela tiver granularidade mensal (year + month), o código usa taxa mensal; se só tiver Company, usa taxa anual. Isso torna a pipeline robusta a variações de schema do dataset cambial.

consolidacao_gaps.py
group_cols = ["Company", "month", "BFPP", "L01"]

actual_agg = actual_df.groupby(group_cols, as_index=False)["Actual_EUR"].sum()
plan_agg   = plan_df.groupby(group_cols, as_index=False)["Plan_EUR"].sum()

# Outer join: captura execução sem orçamento e vice-versa
final_df = actual_agg.merge(plan_agg, on=group_cols, how="outer")
final_df["Actual_EUR"] = final_df["Actual_EUR"].fillna(0)
final_df["Plan_EUR"]   = final_df["Plan_EUR"].fillna(0)

# Gap absoluto e percentual (NaN quando Plan = 0)
final_df["Gap_EUR"] = final_df["Actual_EUR"] - final_df["Plan_EUR"]
final_df["Gap_%"] = np.where(
    final_df["Plan_EUR"] != 0,
    final_df["Gap_EUR"] / final_df["Plan_EUR"],
    np.nan
)

📈 Receita Deflacionada

Deflated Gross Sales Value apresenta gap positivo massivo de +€74.8B porque não havia baseline de plan cadastrado para essa conta. Indica lacuna no processo de planejamento, não necessariamente over-performance real.

📉 Volume Abaixo do Esperado

Toneladas Equivalentes registrou -€18.8B de gap, com Actual de €3.3B contra Plan de €21.7B. Sinal de que volumes físicos ficaram significativamente abaixo do planejado no trimestre.

💱 Impacto Cambial LATAM

As duas companies (2236 e 5454) operam com moedas distintas. A volatilidade cambial da região LATAM amplifica desvios quando a taxa mensal difere do câmbio utilizado no planejamento anual.

🔍 Dados Não Mapeados

A presença de UNMAPPED_L01 indica contas financeiras sem hierarquia definida no fsitem_df. Recomendação: revisão do cadastro mestre de contas para garantir cobertura total.
